# EaseHire Landing Page Prompt

Design a modern, high-converting SaaS landing page for **EaseHire**, a recruitment automation and applicant tracking platform that helps companies streamline hiring from job posting to candidate selection. The page should feel premium, fast, and trustworthy, with a clean SaaS aesthetic inspired by polished modern products, while still feeling slightly futuristic and AI-powered.

The landing page should communicate the core value within **3-5 seconds**: hiring becomes faster, more automated, and easier to manage with one platform. The two conversion goals are **Start Free Trial** and **Book Demo**, with **Start Free Trial** as the visually dominant CTA.

## DESIGN SYSTEM (REQUIRED)

- **Platform:** Web landing page, mobile-first and fully responsive
- **Tech Direction:** React + Tailwind CSS + `shadcn/ui` components + Framer Motion animations
- **Theme:** Light, premium SaaS, minimal, high-clarity, slightly futuristic
- **Background:** Soft white to cool blue gradient base, using subtle mesh or radial gradients
- **Primary Color:** Indigo `#4F46E5` for primary CTAs, links, highlights
- **Accent Color:** Cyan `#06B6D4` for data accents, glow details, charts, and interface emphasis
- **Support Accent:** Violet `#7C3AED` used sparingly for badges or AI labels
- **Text Primary:** Slate `#0F172A`
- **Text Secondary:** Slate `#475569`
- **Borders:** Soft slate `#E2E8F0`
- **Surfaces:** White and lightly tinted glass-style cards with soft shadows and subtle borders
- **Typography:** Bold, modern headings in Inter or Satoshi style; body text clean and readable
- **Layout:** Strong visual hierarchy, wide spacing, generous whitespace, refined grid alignment
- **Motion:** Smooth Framer Motion style reveal animations, staggered section entrances, soft hover states, subtle dashboard float/parallax, no excessive motion

## CONTENT AND POSITIONING

- Target audience: startups, HR teams, and recruitment agencies
- Position EaseHire as an all-in-one hiring operating system that replaces fragmented hiring tools and manual recruiting workflows
- Keep copy concise, sharp, and benefit-focused
- Avoid HR jargon, clutter, and long paragraphs
- Use polished placeholder logos, testimonials, and pricing content that look launch-ready but are easy to replace later

## PAGE STRUCTURE

### 1. Header

- Minimal sticky header with the **EaseHire** logo on the left
- On the right, include only one CTA button: **Start Free Trial**
- Keep navigation minimal or hidden to preserve a focused conversion path

### 2. Hero Section

This is the most important section. It must immediately explain the value of the product.

- Left side:
  - Eyebrow badge such as: `AI-powered hiring automation`
  - Headline: **Hire Top Talent Without the Hiring Chaos**
  - Subheading: **Streamline your hiring process, automate workflows, and manage candidates - all in one powerful platform.**
  - Primary CTA: **Start Free Trial**
  - Secondary CTA: **Book Demo**
  - Small supporting line below CTAs: `No credit card required`
- Right side:
  - A premium dashboard mockup built in UI, not a flat screenshot
  - Show a candidate pipeline with stages like `Applied`, `Screening`, `Interview`, `Offer`, `Hired`
  - Include analytics widgets, hiring funnel metrics, candidate score cards, and AI recommendation indicators
  - Add small visual details like avatars, charts, filters, stage tags, and status chips
- Add soft gradient glow, layered cards, and subtle motion to make the hero feel alive

### 3. Social Proof

- Add a clean social proof strip directly below the hero
- Include:
  - `Trusted by 4,000+ recruiters`
  - `Reduce hiring time by 60%`
  - `3x faster shortlist creation`
- Show a row of polished fictional company wordmarks
- Add a compact avatar cluster or one short testimonial highlight with a measurable result

### 4. Problem to Solution Section

Create a split layout with a clear contrast between current hiring pain and the EaseHire workflow.

- Left column title: `Why hiring still feels broken`
- Show these problems:
  - Too many disconnected hiring tools
  - Manual screening and repetitive admin work
  - Slow hiring cycles that lose strong candidates
  - Limited visibility into team hiring performance
- Right column title: `What changes with EaseHire`
- Show these solutions:
  - Centralized applicant tracking in one workspace
  - Automated workflows for screening, routing, and follow-up
  - AI-powered candidate evaluation and prioritization
  - Real-time analytics for smarter hiring decisions
- Use elegant cards, comparison blocks, or connected visual panels

### 5. How It Works

Create a clean 3-step section with icons and short descriptions.

1. **Post jobs across platforms**  
   Publish openings across multiple job boards and career pages from one place.
2. **Track and screen candidates**  
   Organize applicants, automate filtering, and move the right people forward faster.
3. **Interview and hire faster**  
   Coordinate interviews, collaborate with hiring teams, and close top candidates sooner.

- Use simple line icons or polished product icons
- Keep the section airy and easy to scan

### 6. Features Grid

Create a premium 6-card feature grid with strong iconography and concise copy.

- **Applicant Tracking System** - Manage all candidates in one place
- **Recruitment Automation** - Automate screening and workflows
- **Multi-platform Job Posting** - Reach candidates everywhere
- **Interview Scheduling** - Eliminate back-and-forth emails
- **AI Hiring Insights** - Make smarter hiring decisions
- **Analytics Dashboard** - Track hiring performance in real time

- Cards should feel modern and high-value, with subtle hover states and layered depth

### 7. Use Cases

Create tabs or segmented cards for:

- **Startups**
- **HR Teams**
- **Recruitment Agencies**

Each tab/card should explain how EaseHire helps that audience with short, practical benefits. Examples:

- Startups: move fast with lean teams and faster candidate reviews
- HR Teams: standardize workflows and improve hiring visibility
- Recruitment Agencies: manage multiple roles and clients in one system

### 8. Product Preview Section

Show a larger product showcase section that feels like a premium product tour.

- Include a large candidate pipeline interface
- Show filters, hiring stages, scorecards, and search
- Add an analytics dashboard view with charts, conversion metrics, time-to-hire, and source performance
- Layer the UI preview with floating panels or zoomed detail cards
- This section should reinforce product depth without becoming cluttered

### 9. Testimonials

Add 2 to 3 polished testimonial cards with realistic placeholder names, roles, and outcomes.

Example direction:

- `EaseHire cut our hiring turnaround nearly in half and gave our team one place to run every role.`
- `We automated our first-pass screening and started spending more time with qualified candidates instead of spreadsheets.`
- `The analytics made it obvious where candidates were dropping off, and we fixed bottlenecks within weeks.`

Use professional placeholder identities such as founders, talent leads, and agency directors. Focus on time saved, team alignment, and better hiring outcomes.

### 10. Pricing Section

Create a 3-tier pricing layout with one highlighted plan.

- **Starter**
  - 14-day free trial
  - Up to 3 active roles
  - Basic applicant tracking
  - Workflow templates
  - CTA: `Start Free Trial`
- **Growth** (highlighted)
  - `$149/mo`
  - Unlimited active roles
  - Automation workflows
  - AI candidate scoring
  - Interview scheduling
  - CTA: `Start Free Trial`
- **Enterprise**
  - `Custom pricing`
  - Advanced analytics
  - Team permissions
  - Custom onboarding
  - Priority support
  - CTA: `Book Demo`

Include a visible line such as: **No credit card required for the free trial**

### 11. Final CTA

End with a focused, high-contrast CTA section.

- Headline: **Hiring shouldn't be slow, messy, or manual.**
- Supporting text: a short line about replacing chaos with a faster, automated hiring workflow
- Primary CTA: **Start Free Trial**
- Secondary CTA: **Book Demo** if it supports the layout cleanly

## COMPONENT AND UI GUIDANCE

- Use `shadcn/ui` style components such as Button, Card, Badge, Tabs, Avatar, and Separator
- Keep buttons refined and premium, with the primary button clearly more prominent than the secondary
- Use soft shadows, large radii, subtle borders, and strong spacing rhythm
- Build the dashboard mockups as clean UI compositions using cards, charts, chips, tables, and avatars
- Prefer structured cards and data blocks over decorative illustrations
- Use polished placeholder charts and graphs that reinforce speed, efficiency, and performance

## RESPONSIVENESS

- Mobile-first layout
- On mobile:
  - Stack hero content vertically with copy first and product UI second
  - Convert feature and testimonial grids into one-column or two-column layouts where appropriate
  - Keep CTAs large and easy to tap
  - Preserve whitespace and hierarchy without making the page overly tall or cramped
- On desktop:
  - Use wide, confident spacing and balanced two-column sections
  - Let dashboard previews breathe and feel premium

## ANIMATION GUIDANCE

- Use smooth entrance animations for each section
- Add staggered reveal for cards, feature blocks, and testimonials
- Give hero dashboard elements slight floating motion or layered transitions
- Use hover lift and border glow on important cards
- Keep all animation subtle, crisp, and premium

## IMPORTANT CONSTRAINTS

- Avoid cluttered layouts
- Avoid old-fashioned enterprise UI patterns
- Avoid walls of text
- Avoid generic stock-illustration aesthetics
- Avoid making the page feel like an HR portal; it should feel like a premium modern SaaS product

Tip: Keep the same color tokens, card styling, and spacing rhythm consistent across the entire page so the design feels like one cohesive premium product system.

Make the UI look like a premium SaaS product with smooth animations and strong visual hierarchy.
